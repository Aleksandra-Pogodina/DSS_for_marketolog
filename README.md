# Маркетинговая аналитика — DSS для маркетолога

Десктопное приложение на Python + PySide6, помогающее маркетологу:

1. Загрузить файл с данными (CSV или Excel).
2. Сопоставить столбцы с ролями (каналы, кампании, показы, клики, конверсии, стоимость, возраст, доп. категория и т. п.).
3. Получить агрегированные метрики по каналам/кампаниям: CTR, CPC, CVR, CPA, доли затрат и конверсий.
4. Прочитать готовые рекомендации (DSS) на русском языке: какие каналы масштабировать, какие пересмотреть.
5. Просмотреть интерактивные графики Plotly прямо в окне приложения (легенда переключает ряды).
6. Сохранить отчёт в **DOCX**, включающий таблицу метрик, рекомендации и все графики.

## Интерфейс

Светлая или тёмная тема — переключается кнопкой `☀ Светлая ⇄ 🌙 Тёмная` в шапке
любого окна. Выбор сохраняется через `QSettings` и подхватывается при следующем
запуске. Палитра и стили описаны в одном модуле `app/theme.py`; применяются
глобально через `apply_theme(app)`.

### Главное окно

Поток в четыре карточки сверху вниз:

1. **Загрузка данных** — primary-кнопка «Выбрать файл (CSV / Excel)», статус-бейдж справа.
2. **Сопоставление столбцов** — компактная сетка 3 × N: все 12 ролей видны
   одновременно, никакой вертикальной прокрутки. Подсказка «чего ещё не хватает»
   показывается в реальном времени.
3. **Предпросмотр** — таблица данных на всю ширину карточки.
4. **Запуск анализа** — primary-кнопка `Запустить анализ →`, активируется только
   когда выбраны обязательные поля.

### Окно результатов

- Шапка с заголовком, метаданными прогона, primary-кнопкой `Сохранить отчёт DOCX`
  и кнопкой темы.
- Полоса **KPI-карточек** (число сегментов, суммарные показы/клики/конверсии/затраты,
  средние CTR/CVR).
- Вкладки: **Метрики**, **Рекомендации (DSS)**, **Графики (N)** (дашборд),
  **Доп. категория** (если выбрана), **По возрасту**, **Удалённые строки**
  (если есть).

### Дашборд графиков (без прокрутки)

Слева — список графиков, справа — карточка с **одним активным графиком**,
который занимает всё доступное пространство. Plotly настроен `autosize=True` и
`responsive: True`, поэтому график автоматически перерисовывается при изменении
размера окна. Никакой прокрутки внутри графика.

## Установка

```bash
python3 -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

`PySide6-Addons` подтягивает `QtWebEngine`, необходимый для интерактивного
отображения Plotly. Если его не удалось поставить, приложение остаётся рабочим:
на вкладке графиков покажется PNG-фоллбэк и подсказка как включить интерактивность.

## Запуск

```bash
python main.py
```

## Быстрый тест без GUI

```bash
python smoke_test.py
```

Проверяет: чтение CSV, расчёт метрик и KPI, булевы конверсии, условный funnel
chart, построение Plotly и matplotlib графиков, экспорт DOCX, offscreen-сборку
MainWindow и ResultsWindow, переключение темы, наличие вкладки доп. категории.

## Сопоставление столбцов: роли

Все роли необязательны кроме одной из «Каналы» / «Кампании» и одной числовой:

| Роль | Что это | Пример |
|---|---|---|
| Каналы * | один из обязательных | `channel`, `источник` |
| Кампании * | один из обязательных | `campaign` |
| Показы / Клики / Конверсии | числовые метрики | `impressions`, `clicks`, `conversions` |
| Общая стоимость / Стоимость размещения / Стоимость за клик | стоимостные | `total_cost`, `cpc` |
| Возраст | для возрастной разбивки | `age` |
| Город / Месяц | для агрегатов | `city`, `month` |
| **Доп. категория** | любой категориальный столбец для отдельного среза | `gender`, `product_category`, `region` |

«* — обязательно хотя бы одно».

## Какие метрики считаются

| Метрика | Формула | Требует столбцы |
|---|---|---|
| Показы / Клики / Конверсии | сумма по сегменту | соответствующие столбцы |
| Общая стоимость | сумма по сегменту | total_cost или placement_cost |
| **CTR** | клики / показы | clicks, displays |
| **CPC** | стоимость / клики | clicks + (total_cost ∣ placement_cost ∣ cpc) |
| **CVR** | конверсии / клики | clicks, conversions |
| **CPA** | стоимость / конверсии | conversions + (total_cost ∣ placement_cost) |
| Доля затрат | стоимость сегмента / суммарная | total_cost или placement_cost |
| Доля конверсий | конверсии сегмента / суммарные | conversions |

Метрика не рассчитывается, если выбранных данных недостаточно. Деление на ноль и
пропуски безопасно обрабатываются (результат → NaN, в отчёте — прочерк).

### Булевы конверсии

Если столбец «Конверсии» содержит признаки события вместо чисел, он автоматически
распознаётся и суммируется как «факты конверсии». Поддерживаются:

- `yes` / `no`, `y` / `n`, `true` / `false`, `t` / `f`
- `да` / `нет`, `д` / `н`, `истина` / `ложь`
- `1` / `0`, `+` / `-`
- настоящий Python/numpy `bool`

## Доп. категория (новое)

Роль **«Доп. категория»** позволяет выбрать любой категориальный столбец,
который не покрыт встроенными ролями (пол, регион, категория покупки, сегмент
аудитории и т. п.). Что добавляется:

- отдельная **сводка** по этой категории (`extra_summary` в `AnalysisResult`)
  в разрезе каналов, с теми же базовыми метриками;
- отдельный график в дашборде «{Метрика}: {категория} × Каналы» — bar chart с
  каналами в легенде, который можно скрывать кликом;
- отдельная вкладка **«Доп. категория · {имя}»** в окне результатов с таблицей
  сводки и пояснительной подписью;
- отдельный раздел в DOCX-отчёте с таблицей сводки.

Ключевая метрика выбирается автоматически: `conversions → clicks → displays →
total_cost → placement_cost` — первая доступная.

## Графики

В UI используется **Plotly** (через `QWebEngineView`); в DOCX — matplotlib (PNG).

Что строится при достаточных данных:

- **Сводный funnel chart** — показы / клики / конверсии вместе. Включает только
  те метрики, что реально выбраны. Если выбрано две метрики — название
  «Показы и клики», если три — «Показы, клики и конверсии», если одна — этот
  график не строится вовсе (есть ранжирующий bar).
- **KPI на двух осях** — CTR/CVR (слева, %) и CPC/CPA (справа, ₽).
- **Доли затрат и конверсий** — сравнение бюджета и вклада сегмента.
- **Тепловая карта KPI** — цвет ячейки показывает отклонение от среднего по
  данному KPI в стандартных отклонениях σ, а в клетках выводятся реальные
  значения метрик. Шкала симметрична относительно нуля, легенда подписана
  «Отклонение от среднего, σ».
- **Доп. категория × Каналы** — если выбрана дополнительная категория.
- Ranking bar charts по конверсиям, кликам, показам, стоимости, CTR, CVR, CPA, CPC.
- Матрица корреляций между метриками.
- Распределение по возрастным группам (если выбран «Возраст»).

### Подписи каналов и кампаний на графиках

Когда выбраны и каналы, и кампании одновременно, графики используют
двухуровневую ось (multicategory у Plotly): кампания — верхний уровень,
канал — нижний. Каналы одной кампании идут рядом, кампания подписывается один
раз. Никаких длинных меток «канал | кампания».

## Удаление строк при загрузке

Строка удаляется только если:

1. у неё **отсутствует значение в столбце группировки** (нет «Каналов» / «Кампаний»),
2. **или** у неё **все** выбранные числовые метрики пустые.

Числовые столбцы аккуратно приводятся к числу (`«1 200»` → 1200, текст → NaN);
NaN — это просто «нет значения», а не причина выбрасывать строку.

## Экспорт отчёта

Доступен **только DOCX**. PDF-экспорт удалён сознательно: кириллица и широкие
таблицы давали нестабильный результат. DOCX гарантированно открывается в
Word/LibreOffice/Google Docs и одним кликом конвертируется в качественный PDF.

Содержимое отчёта:

- сводка по сегментам с метриками,
- сводка по дополнительной категории (если выбрана),
- рекомендации и предупреждения,
- все графики PNG.

## Известные ограничения

- В sandbox без X-сервера запускается только не-GUI-логика (через `smoke_test.py`);
  Qt-объекты конструируются под `QT_QPA_PLATFORM=offscreen`.
- Авто-угадывание ролей работает по подстрокам в именах столбцов; роль «Доп. категория»
  пользователь подбирает сам (нельзя предугадать произвольное поле).
- Если `QtWebEngine` недоступен, Plotly заменяется на PNG (приложение остаётся рабочим).
